## 六、标签体系

```typescript
// ══ 优先级（一个用例必有其一）══
@p0          // 核心主流程，CI 门禁必跑
@p1          // 重要功能，回归每日跑
@p2          // 边界场景，发版前跑一次

// ══ 测试分层（冗余标记，方便 grep）══
@smoke       // 等同于 @p0
@edge        // 等同于 @p2

// ══ 业务模块 ══
@auth        // 登录/注册/登出
@users       // 用户管理 CRUD
@products    // 商品管理 CRUD + 分类
@orders      // 订单管理
@cart        // 购物车
@dashboard   // 仪表盘
@settings    // 系统设置
@logs        // 操作日志
@permissions // 权限相关（跨模块）

// ══ 特性标记 ══
@slow        // 慢用例，CI 门禁可跳过
@flaky       // 不稳定用例，不应卡门禁
```

## 七、npm scripts

```json
{
  "scripts": {
    "test": "npx playwright test",
    "test:smoke": "npx playwright test tests/smoke/",
    "test:regression": "npx playwright test tests/smoke/ tests/regression/ --grep '@p0|@p1'",
    "test:edge": "npx playwright test tests/edge/",
    "test:headed": "npx playwright test --headed",
    "test:ui": "npx playwright test --ui",
    "test:debug": "npx playwright test --debug",
    "test:auth": "npx playwright test --grep '@auth'",
    "test:users": "npx playwright test --grep '@users'",
    "test:products": "npx playwright test --grep '@products'",
    "report": "npx playwright show-report"
  }
}
```

### 常用执行命令

```bash
npm run test:smoke                        # 冒烟
npm run test:regression                   # 回归
npm test                                  # 全量
npx playwright test --grep "@p0"          # 只跑 P0
npx playwright test --grep "@p0|@p1"      # P0+P1
npx playwright test --grep "@p0|@p1" --grep-invert "@flaky"  # 跳过不稳定
npx playwright test --grep "@auth" --grep "@p0"               # 只跑登录冒烟
npx playwright test --grep "@p0" --last-failed                # 重跑失败+P0
```

## 八、用例命名规范

采用三段式：**`模块 | 场景 | 预期结果`**

```typescript
// ✅ 推荐
test('登录 | 正确账号密码 | 跳转首页', ...)
test('登录 | 密码错误 | 提示错误信息', ...)
test('用户管理 | 用户名为空 | 提示必填', ...)

// ❌ 避免
test('test1', ...)
test('登录测试', ...)
```

## 九、执行约定

- **每步执行前**：给出完整代码方案，确认后再写入文件
- **代码不执行**：仅给出代码，不运行测试
- **每步产出**：对应的 .ts 代码文件 + README 更新
- **风格**：从零开始独立设计，不参考现有 ui-automation / TestAutomation 代码
